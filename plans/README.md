## Findings considered and rejected (do not re-audit)

- **Vector per-query copies** (`allVectors` `.slice()` per row): by-design-
  adjacent GC cost at personal scale; revisit only if vector recall becomes
  a hot loop.
- **`suggestIdentities` O(N²)**: explicitly owner/host-scheduled batch;
  fine at personal-type cardinality.
- **`created`/`derivations` indexes**: no demonstrated pain; fold into
  whichever v5 migration lands first (012/013/017 build), not standalone.
- **spine.ts split** (history/type-registry extraction): opportunistic
  only — the choke-point coupling is deliberate (I10/I12 by construction).
- **render.ts type-guards, agenda/episode duplication, CLI default
  mismatches, CLI barrel imports**: superseded by plan 002 (CLI removed).
- **`--field` coercion crash, `suggest-identities` cap=0, `--now` lenient
  parse, CLI display gate for quarantined content**: probe-confirmed CLI
  bugs, superseded by plan 002. If the CLI is ever revived from git
  history, fix these first — they are recorded in the audit.
- **Day-anchor scan on every create** (`ensureDayNode` json_extract scan):
  real but sub-ms for years; plan 013's bulk-import design addresses the
  aggravated case; a Ctx-level memo is the cheap fix if it ever shows up
  in a profile.
- **`MemoryError` code taxonomy** (`conflict` carries ~10 meanings): real
  API-design debt, deliberately unplanned this round — it is a breaking
  change that should ride a minor version WITH a consumer; recorded for
  the next audit.
- **Normalized-title lookup seam** (4 divergent implementations + N+1):
  real; deferred because plans 005/007 touch the same regions first and a
  consolidation mid-flight would conflict. Natural follow-up after 011.
- **`edgesOf(id)` public read** (closeEdge needs ids no read supplies):
  folded into plan 012's design as a prerequisite — not separately
  planned.
- **`engines.bun ">=1.2"` floor unverified**: handled inside plan 001 (CI
  matrix leg, continue-on-error).
- **Dead `AuditEntry` export / `isUlid` test-only export**: cosmetic;
  bundle into whichever plan next touches `src/index.ts`.
- **FTS `extra` convention in 3 places**: real drift risk; deferred —
  plan 007 rewrites two of the three sites anyway; re-check after it
  lands.
- **Literal NUL byte in `src/doctor.ts:101`** (found by spike 016,
  verified pre-existing since before the audit): a deliberate dedup-key
  separator written as a raw byte instead of the `" "` escape —
  behavior-identical fix, one byte, but until fixed grep treats the file
  as binary and silently skips it. Fold into whichever plan next touches
  doctor.ts.
- **Test-fixture duplication ×11**: planned — inside 011 (Step 3), not a
  separate plan.
